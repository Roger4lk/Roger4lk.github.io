Specialized Calculator

What We Built

This Project me and my team were tasked with developing a customized calculator for a fictitious product owner. By the end of the development cycles we were left with a fully functional calculator that allowed for unit conversion and creation, and supported relational expressions. It also displayed of intermediate steps for any given calculation as well as the opperation history. This calculator also allowed for saving and loading of the intermediate steps and opperation history. Finally, there was a comprehensive help site explaining each feature.

How We Built It

In a group of five, we applied Agile processes to develop a specialized calculator for our product owner. The focus of this project was to facilitate frequent changes in design requirements while maintaining adaptability within sprints. Using Java Swing for the user interface and some HTML & CSS for the help page, we created an initial functional calculator during the first sprint.

In subsequent sprints, we shifted our efforts towards integrating additional features like the "Intermediate Steps" and "Opperation History" slide out windows, along with additional unit support and other unique features. We also engaged in active negotiations to handle unrealistic acceptance criteria, ensuring that the final product met all requirements within the limited timeframe.

What I Learned

While I gained great experience working with the javax Swing library, the most benificial thing gained from this project is exposer to agile development processes. whether sprint planning meetings to story points and acceptance criteria, gaining experience in this in this process was a view into what development can actually look like.
